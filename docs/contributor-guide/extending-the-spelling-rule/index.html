Extending the Spelling rule

Extending the Spelling rule

Consider extending the Spelling rule in the vale-at-red-hat repository when:

  • Missing words in the Vale dictionary are causing Spelling alerts.

  • Adding the word will benefit to other projects. For example: product and tool names, common terms specific to the software industry.

When the word is too specific to benefit to other projects, consider Creating a vocabulary for a project.

Prerequisites

  • * Installing and configuring Vale

  • Understanding Vale output

  • Defining a Vale onboarding strategy

Procedure

  1. Edit the .vale/styles/RedHat/Spelling.yml file and add the missing word to the filters list. A filter is a case-sensitive regular expression specifying words to ignore during spell checking.

    Excerpt of the Spelling rule
    extends: spelling
message: "Did you really mean '%s'?"
level: error
filters:
  - '\.NET'
  - 'Dev Workspace'
  - 'I/O'
  - "[dD]evfile"
  - "[sS]harding"
  - "Let\'s Encrypt"
  - adoc
  - Ansible
  - Antora
  - API
  - AsciiDoc

  2. Edit the .vale/fixtures/RedHat/Spelling/testvalid.adoc file and add the missing word in a new line.

  3. Edit the test suite for invalid entries: .vale/fixtures/RedHat/Spelling/testinvalid.adoc file and add an undesired form of the missing word in a new line.

  4. Edit the test suite for valid entries: .vale/fixtures/RedHat/Spelling/valid.adoc file and add a valid form of the missing word in a new line.

Verification

  1. Run the vale command on a content file containing the word which was missing.

    $ cd .vale/fixtures/RedHat/Spelling/
$ vale .

  2. Tests are successful when Vale:

    • Reports no error for the testvalid.adoc file

    • Reports an error for each line for the testinvalid.adoc file

  3. Run only the Spelling rule on your content.

    $ vale --config=.vale/fixtures/RedHat/Spelling/ <content files>
